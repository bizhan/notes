# POO & S.O.L.I.D

## Frameworks

### Ventajas

- UNA solución
- Estructura

### Inconvenientes

- GRAN dependencia
- Limita la flexibilidad
- Obsolescencia

## POO & S.O.L.I.D

- Arquitectura emergente
- Crecimiento orgánico
- Desarrollo ágil/evolutivo
- Centrarse en el negocio
- Poder postponer decisiones
- Tomar decisiones conscientemente

## Arquitectura

- Core (domain): OO + SOLID (TDD, BDD, DDD)
- Event bus
- HTTP delivery, CLI delivery
- Database

## Desarrollo negocio

- DDD
- Clean code / SOLID
- TDD
- OO (Alan Kay, message passing)
- Wrapper sobre librerías externas con la funcionalidad usada

## Conclusiones

- Los frameworks son una decisión muy importante y condicionan el proyecto
- Las decisiones tienen que ser conscientes

## Existen alternativas

- Que permiten centrarse en el negocio
- Que facilitan el desarrollo ágil
- Que evolucionan a largo plazo y a buena velocidad
